Flex与Bison

本文作为读《flex与bison》的读书和学习记录，旨在入门编译器相关领域。

flex和bison是用来生成程序的工具，它们所生成的程序能够处理结构化输入。最初的flex和bison是用来生成编译器的，但后来它们被证明在其他很多领域也非常有效。

词法分析与语法分析

最早的编译器可以追溯到上个世纪20年代，这些编译器使用很特别的技术来分析它们所编译的程序源代码的语法。后来，这个领域逐渐获得关注，词法分析成为一个众所周知的领域。

在这个领域中，一个关键的想法是把分析工作分为两个部分：

  • 词法分析：通常所做的事情就是在输入中寻找字符的模式（pattern）
  • 语法分析

由 flex 生成的词法分析器可以读取输入，匹配输入与所有的正则表达式并且执行每次匹配后适当的关联动作。flex 会把所有的正则表达式翻译成一种高效的内部格式（有穷自动机），几乎可以同时处理所有需要的匹配模式，速度非常快。

第一个Flex程序

字数统计程序是 unix 中很常用的一个工具。读取一个文件然后报告这个文件的行数、单词数和字符数。

%{
int chars = 0;
int lines = 0;
int words = 0;
%}
%%
[a-zA-Z]+ { words++; chars += strlen(yytext); }
\n  { lines++; chars++; }
. { chars++; }
%%

int main(int argc, char *argv[] ) {
  yylex();  
  printf("%8d%8d%8d\n", lines, words, chars);
  return 0;
}

翻译程序：通过flex将*.l的felx代码翻译成c代码，然后使用cc lex.yy.c -lfl编译链接

上面的发i吗包含三个部分，各部分用%%分割：

  • 第一部分包含声明和选项设置
  • 第二部分是一些列模式和动作
  • 第三部分则是会被拷贝到生成词法分析器中的 c 代码

Flex与Bison协同

一个简单的桌面计算机程序。首先编写一个词法分析器，然后编写一个语法分析器，并把两者结合起来。

%%
"+"  { printf("PLUS\n"); }
"-"  { printf("MINUS\n"); }
"*"  { printf("TIMES\n"); }
"/"  { printf("DIVIDE\n"); }
"|"  { printf("ABS\n"); }
[0-9]+  { printf("NUMBER %s\n", yytext); }
\n    { printf("NEWLINE\n"); }
[ \t] { }
.   { printf("Mystery character %s\n", yytext); }
%%

作为协同程序的语法分析器

大多数包含flex词法分析的程序使用词法分析器来获得一个记号六，这样可以方便语法分析器的处理。

每当程序需要一个记号的时候，它调用yylex()来读取一小部分输入然后返回相应的记号。当程序需要下一个记号的时候，yylex()会被再次调用。词法分析器以协同程序的方式来运行，也就是说，每当它返回的时候，它会记住当前处理的位置，并重这个位置开始去处理下一次调用。
